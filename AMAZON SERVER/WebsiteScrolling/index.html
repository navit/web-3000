STEP 11

> Log in to the box:

$ssh ubuntu@whatever.amazonaws.com -i ~/Downloads/yourKey.pem

Breaking it down:

> ssh = log into the server but on the command line. Think FTP.
> ubuntu@whatever.amazonaws.com = username@address.com
> ubuntu is automatic default username for a new EC2 Ubuntu server.
> -i ~/Downloads/yourKey.pem = identity file argument.

STEP 13

> Install Apache, a web server software. The engine behind > 100 million web sites.
It’s like Finder for the WWW.
> Type in terminal:


$ sudo apt-get install apache2

Breaking it down:

> sudo = super user “do”. sudo means that the computer is your bitch. (You need permission to sudo. In this case Amazon pre-configured it).
> apt-get = the command line program to run, our package manager
> install = what we want the program to do. Bet you can guess this one.

Step 20

Edit SSHD settings

sudo vi /etc/ssh/sshd_config

Now type:
>“i” to begin editing. (i as in insert)
> Set port from 22 to 30000. This is for security.
> PermitRootLogin should be no
> PasswordAuthentication should be yes
> Hit ESC.
> Type “:wq” to save and quit.
